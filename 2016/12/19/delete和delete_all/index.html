delete_all与destroy_all

2016-12-19 rails 阅读0次 评论

rails中删除多条数据有两种，一种是xxx.delete_all，另一种是xxx.destroy_all

destroy_all

查看源码如下：

# ==== Examples
#
#   Person.destroy_all("last_login < '2004-04-04'")
#   Person.destroy_all(:status => "inactive")
#   Person.where(:age => 0..18).destroy_all
def destroy_all(conditions = nil)
  if conditions
    where(conditions).destroy_all
  else
    to_a.each {|object| object.destroy }.tap { reset }
  end
end

delete_all

# ==== Example
#
#   Post.delete_all("person_id = 5 AND (category = 'Something' OR category = 'Else')")
#   Post.delete_all(["person_id = ? AND (category = ? OR category = ?)", 5, 'Something', 'Else'])
#   Post.where(:person_id => 5).where(:category => ['Something', 'Else']).delete_all
#
# Both calls delete the affected posts all at once with a single DELETE statement.
# If you need to destroy dependent associations or call your <tt>before_*</tt> or
# +after_destroy+ callbacks, use the +destroy_all+ method instead.
def delete_all(conditions = nil)
  raise ActiveRecordError.new("delete_all doesn't support limit scope") if self.limit_value

  IdentityMap.repository[symbolized_base_class] = {} if IdentityMap.enabled?
  if conditions
    where(conditions).delete_all
  else
    statement = arel.compile_delete
    affected = @klass.connection.delete(statement, 'SQL', bind_values)

    reset
    affected
  end
end

结论

通过查看destroy_all和delete_all的源码以及rails c的实例可以知道，destroy_all是通过先找出所有符合条件的语句，然后通过each来一条一条的删除，而delete_all比较简单粗暴，直接通过where条件语句来判断哪些内容是符合的，直接删除。
那么问题来了，既然两条语句都可以达到目的，那么在开发中应该使用哪种呢？

　应该是视情况而定吧。当数据量多的时候，如果使用delete_all，那么可能就会造成锁表（前提是使用MyISAM引擎），当使用destroy_all时，不会造成表锁，但是IO开销就比较大，还是需要根据实际中应用中来决定。
