Ubuntu部署php环境

2016-10-15 ubuntu, php 阅读0次 评论

忙着工作又忘了写日志了，还是得坚持一个月写一篇以上。

前几天测试了一下编译安装php，nginx，mysql，所以就写篇文章总结一下，虽然换工作后都不搞PHP了，但知识还是需要巩固一下。

以下步骤均基于Ubuntu 14.04，在Ubuntu16.04也可以通过。

第一步，安装PHP5.6

1.1 下载php5.6

wget http://211.162.74.230:9011/cn2.php.net/c3pr90ntc0td/distributions/php-5.6.28.tar.gz

1.2 安装依赖包

sudo apt-get install libxml2-dev openssl openssl-dev
如果没有openssl-dev包可以尝试
sudo apt-get install libcurl3-openssl-dev

1.3 进行编译安装

sudo ./configure --prefix=/usr/local/server/php --with-config-file-path=/usr/local/server/php --enable-mbstring --enable-ftp --with-mysql=mysqlnd --with-mysqli=mysqlnd --with-openssl-dir=/usr/bin/openssl --with-openssl --with-pdo-mysql=mysqlnd --with-pear --enable-sockets--with-freetype-dir=/usr --enable-gd-native-ttf --with-zlib--with-libxml-dir=/usr --with-xmlrpc --enable-zip --enable-fpm --enable-fpm--enable-xml --enable-sockets --with-zlib --with-iconv --enable-zip--with-freetype-dir=/usr/lib/ --enable-soap --enable-pcntl --enable-cli

1.4 发现的问题

提示不存在zlib..
然后apt-get install zlib..竟然发现没有这个包..
好吧，换种方式
wget http://211.162.127.24/files/3045000000170B3C/sourceforge.net/projects/libpng/files/zlib/1.2.8/zlib-1.2.8.tar.gz
tar -zxvf zlib-1.2.8.tar.gz
sudo ./configure
sudo make && make install

进行编译安装

sudo make && make install

稍等片刻，出去喝杯咖啡~

OK.安装完成..

执行命令：php –v。如果查询不到php版本信息，则执行命令： sudo cp /usr/local/server/php/bin/php /usr/bin/php

第二步，Nginx(1.11.6) + php-fpm

sudo apt-get install nginx
cd /etc/nginx/sites-enabled
vim default
#找到这段话，并打开注释
location ~ \.php$ {
        fastcgi_split_path_info ^(.+\.php)(/.+)$;
        # NOTE: You should have "cgi.fix_pathinfo = 0;" in php.ini

        # With php5-cgi alone:
        fastcgi_pass 127.0.0.1:9000;
        # With php5-fpm:
        #fastcgi_pass unix:/var/run/php5-fpm.sock;
        fastcgi_index index.php;
        include fastcgi_params;
}
sudo service nginx restart

新建一个php文件测试..

vim /usr/share/nginx/html/hello.php
#编辑如下

访问:[http://192.168.180.130/hello.php](http://192.168.180.130/hello.php)

#报错
502 Bad Gateway

查看日志 /var/log/nginx/error.log

2016/11/23 23:05:30 [crit] 38909#0: *1 connect() to unix:/var/run/php5
-fpm.sock failed (2: No such file or directory) while connecting 
to upstream, client: 192.168.180.1, server: localhost, request: "GET 
/hello.php HTTP/1.1", upstream: "fastcgi://unix:/var/run/php5-fpm.sock:"
, host: "192.168.180.130"

大致意思就是 php5-fpm位置不对.. 回到最开始解压php-5.6.28的位置

cd ~/php-5.6.28 
sudo cp sapi/fpm/init.d.php-fpm /etc/init.d/php-fpm
sudo chmod +x /etc/init.d/php-fpm
cd /usr/local/server/php/etc
sudo cp php-fpm.conf.default php-fpm.conf

sudo vim /usr/local/server/php/etc/php-fpm.conf

25行
;pid = run/php-fpm.pid
修改为
pid = run/php-fpm.pid

149行
user = nobody
group = nobody
修改为
user = young #即当前用户..
group = young #即当前用户组..

#启动php-fpm
sudo /etc/init.d/php-fpm start
Starting php-fpm  done

再次访问:http://192.168.180.130/hello.php 终于出现久违的phpinfo()页面啦~

还没结束呢.. 继续安装 mysql、redis、git sudo apt-get install mysql-server-5.6 mysql-client-5.6 git redis-server

第三步，安装Composer

只需要4条语句

curl -sS https://getcomposer.org/installer | php

/usr/bin/php composer.phar –version

sudo mv composer.phar /usr/local/bin/composer

查看是否安装成功..

composer -version

更改composer源..

composer config -g repo.packagist composer [https://packagist.phpcomposer.com](https://packagist.phpcomposer.com)

查看是否更改成功

composer config -l -g

第四步，安装各种php扩展

举个栗子，要安装curl

#进入到解压缩的php目录
cd php-5.6.28
#复制php.ini 到php安装的目录
sudo cp php.ini.production /usr/local/server/php/php.ini

#进入php扩展包界面
cd ext/curl
#编译
/usr/local/server/php/bin/phpize
#执行
./configure --with-php-config=/usr/local/server/php/php-config
#安装
sudo make
sudo make install
#加入扩展包
vim /usr/local/server/php/php.ini
#增加以下语句
extension=curl.so
